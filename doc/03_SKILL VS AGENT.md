
### SKILL VS AGENT

**Skill = 可复用的知识包**
Agent 是角色（谁来干）
Skill 是能力（怎么干）

| 维度  | Agent（角色）        | Skill（能力）              |
| --- | ---------------- | ---------------------- |
| 定义  | `agents/*.md` 文件 | `skills/*/SKILL.md` 文件 |
| 回答  | 「我是谁？」           | 「怎么干？」                 |
| 内容  | ** 身份、职责、行为准则 ** | ** 步骤、标准、输入输出 **       |
| 类比  | 岗位说明书            | 操作手册 / SOP             |

### SOP → Skill → 组织智能：三级演进

**Level 1：SOP（可执行）**
- 单个 `SKILL.md` + 固定步骤 → 完成一类标准任务
- 例：代码审查按 5 条固定规则检查


**Level 2：专家系统（可扩展）**
- `SKILL.md` + `references/` + `templates/` + `scripts/`
- 处理整个领域的 20+ 种变体
- 例：代码审查覆盖安全、性能、可维护性、API 规范


**Level 3：组织智能（可规模化）**
- 多个 SubAgent + 各自专属 Skills + 流水线协作
- 例：分析 Agent → 审核 Agent → 测试 Agent → 部署 Agent


### Memory + Sub-Agents + Skills

| SDD 层级 | 你写的文件 | 回答的问题 |
|----------|-----------|-----------|
| 项目规范 | `AGENTS.md` | 项目是什么？怎么做？ |
| 角色规范 | `agents/*.md` | 谁来干？权限边界？ |
| 能力规范 | `skills/*/SKILL.md` | 具体怎么干？步骤？ | 


### 工具分层
工具分成四层去看。
*** "原子技能" ***：代表是 grill-me、tdd skill 这些——一个 SKILL.md 就六行，功能极其单一，比如"在 AI 写代码前逼它先追问"。
*** "上下文持久化" ***： 代表是 AGENTS.md、Trellis 这些——解决多次会话 AI 忘了项目规矩的问题。
*** "规范框架" ***：代表是 OpenSpec、Spec-Kit——管的是 spec 这份文档本身怎么生成、怎么版本化、怎么 verify。
*** "完整方法论" ***：代表是 Superpowers 和 BMAD——它们不是管某一件事，而是接管你从头脑风暴到代码合入的整个流程。

### 怎么判断自己是哪一层
怎么判断自己是哪一层？看你当下最痛的是什么。
如果痛的是"AI 不问我就开始瞎写"，你要的是 L1。
如果痛的是"这次会话教给 AI 的项目规矩，下次它又忘了"，你要的是 L2。
如果痛的是"需求和代码中间那段——设计方案、任务拆分、验收标准——每次都乱七八糟没法复用"，你要的是 L3。
如果痛的是"我希望从想一个功能到合入 main 之间，整个流程是规范的、自动化的、可审计的"，那才是 L4
